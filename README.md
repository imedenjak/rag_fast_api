# RAG API 🤖

A Retrieval-Augmented Generation (RAG) API built with LangChain, Qdrant, and FastAPI, containerized with Docker.

## Overview

This project loads content from a web page, splits it into chunks, stores embeddings in **Qdrant** vector store, and exposes a FastAPI endpoint to answer questions based on the retrieved context using OpenAI's GPT-3.5-turbo.

Indexing and serving are **separated** — run ingestion once locally to build the vector store, then Docker mounts the folder and serves it.

## Tech Stack

- **LangChain** — RAG pipeline orchestration
- **Qdrant** — Persistent local vector store
- **OpenAI** — Embeddings (text-embedding-ada-002) + LLM (GPT-3.5-turbo)
- **FastAPI** — REST API
- **Docker** — Containerization

## Getting Started

### 1. Clone the repository

```bash
git clone <your-repo-url>
cd my_rag
```

### 2. Set up environment variables

```bash
# Copy the example env file
cp .env.example .env

# Edit .env and add your OpenAI API key
OPENAI_API_KEY=sk-xxx
```

### 3. Install dependencies locally

```bash
pip install -r requirements.txt
```

### 4. Run ingestion — build Qdrant index (once only)

```bash
cd app
python ingest.py
```

Expected output:
```
Loading documents...
Loaded 1 documents
Splitting documents...
Created 63 chunks
Embedding and saving to Qdrant...
Saved 63 vectors to Qdrant
Ingestion complete!
```

> This step fetches the web page, splits it into chunks, generates OpenAI embeddings, and saves them to `qdrant_db/`. Only needs to be run once — or when you want to re-index new content.

### 5. Start with Docker

```bash
docker compose up --build -d
```

### 6. Check logs

```bash
docker compose logs -f api
```

Wait until you see:
```
RAG chain ready!
```

## API Usage

### Swagger UI (Recommended)

```
http://localhost:8000/docs
```

### Health Check

```bash
# Windows
curl.exe http://localhost:8000/health

# Linux / Mac
curl http://localhost:8000/health
```

Response:

### Ask a Question

**Windows PowerShell:**
```bash
curl.exe -X POST http://localhost:8000/ask -H "Content-Type: application/json" -d "{\"question\": \"What is Task Decomposition?\"}"
```

**Linux / Mac:**
```bash
curl -X POST http://localhost:8000/ask \
  -H "Content-Type: application/json" \
  -d '{"question": "What is Task Decomposition?"}'
```

Response:
```json
{
  "question": "What is Task Decomposition?",
  "answer": "Task decomposition is the process of breaking down a complex task into smaller, more manageable steps..."
}
```

### Python Client

```python
import requests

response = requests.post(
    "http://localhost:8000/ask",
    json={"question": "What is Task Decomposition?"}
)
print(response.json()["answer"])
```

## When to Re-run ingest.py

```bash
cd app
python ingest.py
```

Re-run ingestion when you want to:
- Index a different or updated web page
- Change chunk size or overlap settings
- Switch embedding models (must delete qdrant_db/ first)
- Rebuild the vector store from scratch

> `recreate_collection` in ingest.py clears old vectors before re-indexing — no duplicates.

## Reset Vector Store

```bash
# Windows PowerShell
Remove-Item -Recurse -Force .\app\qdrant_db\

# Then re-ingest
cd app
python ingest.py
```

## Architecture

```
┌─────────────────────────────────────────┐
│  ingest.py (run once locally)           │
│                                         │
│  Web Page → Split → Embed → Qdrant      │
└─────────────────┬───────────────────────┘
                  │ app/qdrant_db/ folder
┌─────────────────▼───────────────────────┐
│  Docker Container (FastAPI)             │
│                                         │
│  Startup: load Qdrant (2-3 seconds)     │
│  Request: retrieve → prompt → LLM       │
└─────────────────────────────────────────┘
```

## Why Qdrant over pgvector

| | pgvector | Qdrant |
|--|---------|--------|
| **Setup** | ❌ Postgres + Docker | ✅ Just a folder |
| **Docker services** | 2 (api + db) | 1 (api only) |
| **Version conflicts** | ✅ Frequent | ❌ None |
| **Data location** | Docker volume | `qdrant_db/` folder |
| **Reset** | `docker compose down -v` | Delete folder |
| **Production** | ✅ | ✅ Qdrant Cloud |

## Notes

- Qdrant data lives in `app/qdrant_db/` — a plain folder mounted into Docker.
- No database service needed — simpler `docker-compose.yml` with one service.
- OpenAI embeddings use `text-embedding-ada-002` with **1536 dimensions**.
- Question must be between 3 and 500 characters.
- Re-ingestion required when switching embedding models — dimensions must match.
